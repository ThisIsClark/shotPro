# 手脚协调评估功能

## 功能概述

新增了"手脚协调"评估维度，专门检测投篮中常见的"手快脚慢"问题。

## 什么是"手快脚慢"？

**定义**：腿部还在下蹲或刚开始伸展时，手部已经开始向上举球。

**问题**：
- 上下身动作脱节
- 腿部力量无法有效传递到手部
- 影响投篮稳定性和准确性

**正确的协调**：
- 腿部发力和手部举球应该同步
- 或者先腿后手（腿部先启动发力）
- 形成流畅的"由下至上"的发力链

## 技术实现

### 1. 新增问题类型

在 `IssueType` 枚举中添加：
```python
HAND_FAST_FOOT_SLOW = "hand_fast_foot_slow"
```

### 2. 新增评估维度

在 `RULES` 字典中添加：
```python
"hand_foot_coordination": {
    "name": "手脚协调",
    "name_en": "Hand-Foot Coordination",
    "description": "手部举球与腿部发力的协调性",
    "weight": 根据投篮方式动态调整
}
```

**权重分配**：
- **一段式投篮**：6%（从肘部和平衡各减3%）
- **二段式投篮**：4%（协调性相对不那么重要）

### 3. 检测算法

**分析时间窗口**：
- 准备阶段末尾 5 帧
- 上升阶段开始 10 帧
- 共约 15 帧的过渡期

**关键指标**：
1. **手腕高度变化**：
   - 检测手腕 Y 坐标减小（向上移动）
   - 阈值：`wrist_delta < -0.015`

2. **膝盖角度变化**：
   - 检测膝盖角度增大（腿伸展）
   - 阈值：`knee_delta > 3.0°`

**判断逻辑**：
```
coordination_gap = hand_lift_start - leg_extend_start

if coordination_gap > 5:
    # 手比腿早 5+ 帧 → 严重手快脚慢
    severity = HIGH
    score = max(40, 100 - gap * 5)
    
elif coordination_gap > 2:
    # 手比腿早 2-5 帧 → 轻微手快脚慢
    severity = MEDIUM
    score = max(70, 100 - gap * 5)
    
elif -2 <= coordination_gap <= 2:
    # 基本同步 → 优秀
    score = 100
    
else:  # coordination_gap < -2
    # 腿先发力 → 很好
    score = 95
```

## 评估反馈

### 优秀（≥90分）
✓ 手脚协调优秀，腿部发力和手部举球同步良好。力量传递顺畅，符合由下至上的发力链原则

### 一般（70-89分）
△ 手脚协调基本合理，但有轻微的不同步。建议感受腿部向上蹬地的力量，让手臂跟随腿部发力

### 需改进（<70分）
✗ 存在明显的手快脚慢问题：腿还在下蹲时手就开始上举。这会导致上下身脱节，影响力量传递。应该先腿部发力，手部跟随

## 改进建议

当检测到手快脚慢问题时，系统会提供：

**高优先级建议**：
- **标题**：改善手脚协调
- **描述**：投篮时注意手脚协调：应该先感受腿部向上发力，然后手部跟随举球
- **具体练习**：
  1. 下蹲时球保持在胸前低位
  2. 腿部开始向上蹬地
  3. 同时或稍后手臂开始上举
  4. 避免腿还在蹲时手就开始举球

**专业依据**：
- 专业教练强调：投篮是一个由下至上的发力链
- 腿部力量应该传导到手部
- 手快脚慢会导致上下身脱节，影响力量传递和出手稳定性

## 使用方式

功能自动启用，无需额外配置。在每次分析中：

1. 系统自动检测准备→上升阶段的过渡期
2. 分析手腕高度和膝盖角度的变化时序
3. 计算手脚启动的时间差
4. 给出协调性评分和具体建议
5. 如果存在问题，在"改进建议"中高优先级显示

## 测试建议

重新分析之前的投篮视频，查看：
- "各维度评分"中是否新增了"手脚协调"一栏
- 如果视频中存在手快脚慢问题，是否能正确检测并给出建议
- 评分和反馈是否合理
